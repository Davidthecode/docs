title: "Get Invoice"
description: "Retrieve a single invoice by its unique identifier."
openapi: "GET /invoices/{id}"
---

<RequestExample>

```bash cURL
curl -X GET "https://eqs-platform-api.onrender.com/v1/invoices/64a8d1ccdff3e72ccc4a2983" \
  -H "Authorization: API-Key YOUR_API_KEY"
```

You can also specify the fields to be returned using the `fields` query parameter.

```bash cURL
curl -X GET "https://eqs-platform-api.onrender.com/v1/invoices/64a8d1ccdff3e72ccc4a2983?fields=invoiceNumber,amount,status" \
  -H "Authorization: API-Key YOUR_API_KEY"
```

```javascript Node.js
import fetch from "node-fetch";

const invoiceId = "64a8d1ccdff3e72ccc4a2983";
const params = new URLSearchParams({ fields: "invoiceNumber,amount,dueDate" });

fetch(
  `https://eqs-platform-api.onrender.com/v1/invoices/${invoiceId}?${params.toString()}`,
  {
    headers: {
      Authorization: `API-Key ${process.env.EQUIS_API_KEY}`,
    },
  }
)
  .then((res) => res.json())
  .then((json) => console.log(json))
  .catch((err) => console.error("Error:", err));
```

</RequestExample>

### Available Invoice Fields

The following fields can be requested when fetching an invoice:

- invoiceNumber
- amount
- dueDate
- status
- outstandingBalance
- invoiceType
- description
- items
- case
- customerAgreement
- isDue
- salesOrder
- category
- tax
- discount
- statusNote
- autoConvertToCase

### Populated References

The following related resources are automatically populated if part of the selected fields:

- case
- company
- customerAgreement
- salesOrder
