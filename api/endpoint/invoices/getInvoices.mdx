<RequestExample>

```bash cURL
curl -X GET "https://eqs-platform-api.onrender.com/v1/invoices" \
  -H "Authorization: API-Key YOUR_API_KEY"
```

You can also specify the fields to be returned using the `fields` query parameter.

```bash cURL
curl -X GET "https://eqs-platform-api.onrender.com/v1/invoices/64a8d1ccdff3e72ccc4a2983?fields=invoiceNumber,amount,status" \
  -H "Authorization: API-Key YOUR_API_KEY"
```

```javascript Node.js
import fetch from "node-fetch";

const params = new URLSearchParams({
  page: 1,
  limit: 15,
  fields: "invoiceNumber,amount,dueDate",
});

fetch(`https://eqs-platform-api.onrender.com/v1/invoices?${params.toString()}`, {
  headers: {
    Authorization: `API-Key ${process.env.EQUIS_API_KEY}`,
  },
})
  .then((res) => res.json())
  .then((json) => console.log(json))
  .catch((err) => console.error("Error:", err));
```

### Example Error Responses

Invalid limit:

```json
{
  "status": 400,
  "code": "ERR_LIMIT_EXCEEDED",
  "message": "Limit cannot exceed 50"
}
```

Invalid page or limit:

```json
{
  "status": 400,
  "code": "ERR_INVALID_LIMIT",
  "message": "Limit must be a positive integer"
}
```

Invalid query params:

```json
{
  "status": 400,
  "code": "BAD_REQUEST",
  "message": "Invalid query parameter(s): sort"
}
```

### Available Invoice Fields

The following fields can be requested when fetching an invoice:

- invoiceNumber
- amount
- dueDate
- status
- outstandingBalance
- invoiceType
- description
- items
- case
- customerAgreement
- isDue
- salesOrder
- category
- tax
- discount
- statusNote
- autoConvertToCase

### Populated References

The following related resources are automatically populated if part of the selected fields:

- case
- company
- customerAgreement
- salesOrder
