<RequestExample>

```bash cURL
# Get case metrics for the past 90 days
curl -X GET "https://api.equisettle.com/v1/metrics/cases?timeRange=90" \
  -H "Authorization: API-Key YOUR_API_KEY"
```

```bash cURL
# Get workflow metrics for the current year
curl -X GET "https://api.equisettle.com/v1/metrics/workflow?timeRange=thisYear" \
  -H "Authorization: API-Key YOUR_API_KEY"
```

```javascript Node.js
import fetch from "node-fetch";

const metricType = "cases"; // or "cases-volume", "customer-segments", "workflow", "engagements", "risk"
const params = new URLSearchParams({ timeRange: "90" });

fetch(
  `https://api.equisettle.com/v1/metrics/${metricType}?${params.toString()}`,
  {
    headers: {
      Authorization: `API-Key ${process.env.EQUIS_API_KEY}`,
    },
  }
)
  .then((res) => res.json())
  .then((json) => console.log(json))
  .catch((err) => console.error("Error:", err));
```

</RequestExample>

## Query Parameters

<ParamField query="timeRange" type="string">
  Specifies the time range for the metrics. Valid values are `"30"`, `"90"`,
  `"365"`, `"thisYear"`, or any positive number representing days.
</ParamField>
<Note>
  {" "}
  Only the `timeRange` query parameter is allowed. Any other query parameters will
  result in a `400 Bad Request` error.{" "}
</Note>

## Available Metric Types

- `/cases` – Basic case metrics
- `/cases-volume` – Monthly case volume
- `/customer-segments` – Customer segmentation metrics
- `/workflow` – Workflow metrics
- `/engagements` – Team and case engagement metrics
- `/risk` – Financial, overdue, and dispute risk metrics
